gosentry notes for `brrrsentry`

Main reference

- `third_party/gosentry/README.md`

Main features we care about

- struct-aware fuzzing
- integer overflow detection
- panic on selected functions
- LibAFL-backed fuzzing
- race condition catching
- goroutine leak catching
- hang confirmation
- grammar-based fuzzing with Nautilus
- coverage report generation from the campaign corpus

Important gosentry notes for this project

- Grammar mode works best with a single `[]byte` or `string` fuzz input
- Differential fuzzing still converts grammar-generated bytes into domain values inside the harness
- Panic-on-call should be used for critical error paths when it fits the target
- Race and leak replay loops can be enabled with `--catch-races=true` and `--catch-leaks=true`
- Coverage replay is available with `--generate-coverage`
